# 部署文档 - 1919ys 爬虫项目

本文档用于指导将爬虫项目部署到生产环境，并设置自动化的全量或增量爬取任务。

## 1. 项目简介

本项目是一个针对 `1919ys.com` 影视网站的爬虫，旨在抓取其所有分类（电影、电视剧、综艺、动漫、短剧）的播放资源。

*   **目标网站**: `https://www.1919ys.com`
*   **面向用户**: 中文用户（简繁体），主要覆盖中国大陆、港台及海外华人地区。
*   **内容类型**: 影视剧集、综艺节目、动漫番剧等。
*   **输出数据**: 包含播放链接、海报、简介等元数据的 CSV 文件。

## 2. 环境要求

*   **操作系统**: Linux (推荐 Ubuntu 20.04+) / Windows
*   **Python 版本**: Python 3.8+
*   **硬件建议**: 
    *   CPU: 2核+ (推荐 4核+ 以利用多进程)
    *   内存: 4GB+
    *   网络: 稳定的公网连接，建议使用海外服务器以避免潜在的访问限制。

## 3. 部署步骤

### 3.1. 获取代码

将项目代码上传至服务器：

```bash
# 假设上传到了 /opt/spider 目录
cd /opt/spider
```

### 3.2. 安装依赖

```bash
pip3 install -r requirements.txt
```

### 3.3. 首次全量爬取

在首次部署时，建议执行一次全量爬取以建立完整的数据库。

```bash
# 使用 nohup 后台运行，日志输出到 spider.log
nohup python3 main.py > spider.log 2>&1 &

# 查看日志
tail -f spider.log
```

**注意**: 全量爬取可能耗时较长（数小时），请耐心等待。完成后会自动在 `output/` 目录生成 CSV 文件。

## 4. 每日增量更新

为了保持数据最新，建议设置每日定时任务执行增量爬虫。增量爬虫会扫描列表页，直到连续 3 页没有发现新内容为止，效率较高。

### 4.1. 手动测试增量脚本

```bash
python3 incremental.py
```

### 4.2. 设置 Crontab 定时任务

编辑 crontab 配置：

```bash
crontab -e
```

添加以下行（例如每天凌晨 3 点执行）：

```cron
0 3 * * * cd /opt/spider && /usr/bin/python3 incremental.py >> incremental.log 2>&1
```

## 5. 项目结构说明

*   `main.py`: **全量爬虫入口**。使用多进程+多线程架构，适合初始化数据。
*   `incremental.py`: **增量爬虫入口**。适合日常更新，扫描到无新内容自动停止。
*   `spider.py`: 爬虫核心逻辑类。
*   `database.py`: 数据库初始化与连接管理 (SQLite)。
*   `exporter.py`: 数据导出工具 (CSV)。
*   `output/`: 存放导出的数据文件。
*   `vs_1919_flat.db`: SQLite 数据库文件。

## 6. 维护与故障排查

*   **日志**: 检查 `spider.log` 或 `incremental.log` 查看运行状态。
*   **反爬虫**: 如果发现大量 Connection Refused 或 403 错误，可能是 IP 被封禁。建议降低并发数（修改代码中的 `max_workers`）或使用代理池。
*   **数据库**: 数据库使用 SQLite WAL 模式，支持并发写入。如遇数据库锁死，可尝试重启进程或检查是否有僵尸进程持有锁。
*   **导出失败**: 如果数据量过大导致 CSV 导出慢，脚本已优化为分块导出，请确保磁盘空间充足。

## 7. 导出数据说明

导出的 CSV 文件包含以下核心字段：
*   `category`: 分类
*   `title`: 剧名
*   `episode_name`: 集数
*   `play_url`: 播放链接 (唯一标识)
*   `poster_url`: 海报链接
*   `detail_url`: 原网页链接

该数据结构为“宽表”模式，即每一行都是一个可播放的独立资源，方便直接导入搜索引擎（如 Elasticsearch）。
