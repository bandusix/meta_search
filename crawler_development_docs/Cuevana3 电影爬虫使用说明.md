# Cuevana3 电影爬虫使用说明

## 项目简介

这是一个用于爬取 [Cuevana3.top](https://cuevana3.top) 网站电影数据的 Python 爬虫脚本。该脚本支持爬取指定年份或年份范围的电影信息，包括电影标题和 URL，并自动保存为 CSV 格式。

## 功能特点

- ✅ **单年份爬取**：爬取指定单个年份的所有电影数据
- ✅ **年份范围爬取**：支持爬取多个年份的数据（例如：2020-2025）
- ✅ **倒序优先级**：默认从最新年份开始爬取（2025 → 2024 → 2023...）
- ✅ **自动分页处理**：自动识别并遍历所有分页
- ✅ **健壮的异常处理**：包含请求超时、404 错误等异常处理
- ✅ **反爬虫机制**：
  - 随机 User-Agent 请求头
  - 可配置的随机延迟时间（默认 1-3 秒）
  - 模拟人类浏览行为
- ✅ **CSV 输出**：自动保存为 CSV 格式，包含年份、标题和 URL

## 使用方法

### 基本语法

```bash
python cuevana3_scraper.py <起始年份> [结束年份] [选项]
```

### 使用示例

#### 1. 爬取单个年份
```bash
# 爬取 2024 年的所有电影
python cuevana3_scraper.py 2024
```

输出文件：`cuevana3_2024.csv`

#### 2. 爬取年份范围（倒序，从最新开始）
```bash
# 爬取 2020-2025 年的电影，按倒序：2025 → 2024 → 2023 → 2022 → 2021 → 2020
python cuevana3_scraper.py 2020 2025
```

输出文件：`cuevana3_2020-2025.csv`

#### 3. 爬取年份范围（正序）
```bash
# 爬取 2020-2025 年的电影，按正序：2020 → 2021 → 2022 → 2023 → 2024 → 2025
python cuevana3_scraper.py 2020 2025 --no-reverse
```

#### 4. 指定输出文件名
```bash
# 自定义输出文件名
python cuevana3_scraper.py 2024 -o my_movies.csv
```

#### 5. 自定义延迟时间
```bash
# 设置延迟范围为 2-4 秒
python cuevana3_scraper.py 2024 --delay 2 4
```

#### 6. 组合使用
```bash
# 爬取 2023-2025 年，倒序，自定义文件名和延迟
python cuevana3_scraper.py 2023 2025 -o recent_movies.csv --delay 1.5 3
```

### 命令行参数说明

| 参数 | 类型 | 必需 | 说明 | 默认值 |
|------|------|------|------|--------|
| `start_year` | 整数 | 是 | 起始年份（如果只提供一个年份，则只爬取该年份） | - |
| `end_year` | 整数 | 否 | 结束年份（用于爬取年份范围） | None |
| `-o, --output` | 字符串 | 否 | 输出 CSV 文件名 | `cuevana3_{year}.csv` 或 `cuevana3_{start}-{end}.csv` |
| `--delay` | 浮点数 | 否 | 请求延迟范围（秒），需提供两个值：最小值和最大值 | `1 3` |
| `--no-reverse` | 标志 | 否 | 不使用倒序爬取（默认从最新年份开始） | False（默认倒序） |
| `-h, --help` | 标志 | 否 | 显示帮助信息 | - |

## 技术实现细节

### 数据提取方法

脚本使用 **BeautifulSoup** 解析 HTML，通过 CSS 选择器定位电影条目：

1. **定位电影条目**：`div[id^="post-"]`
2. **提取 URL**：`a[href*="/pelicula/"]` 的 `href` 属性
3. **提取标题**：`div.Title` 的文本内容（自动移除 "PELÍCULA" 等标签）

### 分页处理

- 自动从第一页的分页导航中提取总页数
- 构建分页 URL：`https://cuevana3.top/estreno/{year}/page/{page_number}/`
- 遍历所有页面直到完成

### 反爬虫策略

1. **随机 User-Agent**：从预设列表中随机选择
2. **随机延迟**：每次请求后等待 1-3 秒（可配置）
3. **会话保持**：使用 `requests.Session()` 保持连接
4. **异常处理**：捕获超时、404 等错误并继续执行

## 常见问题

### 1. 为什么爬取速度很慢？
为了避免触发反爬虫机制，脚本默认在每次请求后延迟 1-3 秒。您可以通过 `--delay` 参数调整延迟时间，但不建议设置过短。

### 2. 如何处理爬取失败的页面？
脚本会自动跳过失败的页面并继续爬取，失败信息会在终端显示。您可以查看输出日志确认哪些页面失败。

### 3. CSV 文件中有重复数据怎么办？
这可能是网站本身的数据问题。您可以使用以下命令去重：
```bash
# 保留表头，去除重复行
awk '!seen[$0]++' input.csv > output.csv
```

### 4. 如何中断正在运行的爬取？
按 `Ctrl+C` 可以安全中断爬取。

### 5. 爬取的数据可以用于什么用途？
数据仅供个人学习和研究使用，请遵守网站的使用条款和相关法律法规。
